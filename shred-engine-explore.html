Shred Engine

My custom game engine made with C++, Jolt pysics, Box2D

In-Progress

ABOUT THE ENGINE

SHRED is a cross-platform C++ 3D/2D engine in development for over two years, with performance, flexibility, and ease of use in mind. Inspired by the Unity3D engine, with the scripting API and usage freedom I always dreamed of.

CROSS-PLATFORM CORE

Currently runs on OpenGL/ES, but its rendering backend is designed to support APIs such as Vulkan, Direct3D, Metal, or any other required by the target platform. It includes full cross-platform input handling and an audio system that works seamlessly across all targets. The editor is available on both Windows and macOS, and supports building final executables for Windows and Android (via NDK).

RENDERING & PHYSICS

For physics, the engine integrates both Jolt Physics for 3D simulations and Box2D for 2D. Its custom renderer handles both dimensions and is being optimized for high performance and scalability, whether you're building a fast-paced platformer or a 3D world.

FILESYSTEM & BUILD OUTPUT

An import system ensures that textures, audio, and meshes are automatically converted into optimized custom formats for faster in-game and in-editor loading. Games built with the engine export clean and portable binaries. Only the executable and a .shr data package are needed to run a game. The .shr file is a custom compressed and encrypted format that contains all game assets.

IN-EDITOR WORKFLOW

The editor features script hot reloading and a real-time in-editor play mode. Developers can instantly preview gameplay changes without restarting. A object editor is available, powered by the RTTR reflection library, allowing for efficient editing and debugging of game objects and assets directly in the scene.

The editor includes a set of tools designed to improve the development workflow and content creation process:

  • A sprite atlas manager allows for precise control over tiles, ideal for building 2D worlds and managing spritesheet animations.
  • LDtk 2D level editor files are fully supported, so designers can build levels externally and import them into the engine with zero friction.
  • For showcasing and debugging, the engine also features a built-in video recorder powered by ffmpeg. It captures gameplay frames directly from the renderer, producing high-quality, lossless footage ideal for trailers, playtesting, or sharing progress without compromising visual fidelity.
